# 2026-06-15 — image bump 25.11-py3 → 26.05-py3

**Decision:** promote.
**Scope:** `infra/vllm/autoresearch/docker-compose.yml`.
**Tickets resolved:** chipi/agentic-ai-homelab#2 (closed via promote), chipi/agentic-ai-homelab#1 (closed via obsoletion), chipi/agentic-ai-homelab#3 (closed via README vendor-watch note).
**Commit:** `b2957f6`.

## Why we evaluated

`EVAL_HYBRID_ROUTING_2026_06.md` § Summary in `podcast_scraper` cites
`nvcr.io/nvidia/vllm:26.05-py3` as the autoresearch baseline image; this
stack was pinned at `25.11-py3` (inherited from `coder-next/` genesis,
~6 months older than the cited baseline). Any measurement against the
cited eval baseline carried a silent serving-engine drift until we
either bumped or documented the gap. This was the bump test.

## Methodology

Single-GPU GB10 (DGX-class), same model `Qwen/Qwen3-30B-A3B-Instruct-2507`
on both images, same canned prompt (`"Briefly explain how a vector
database stores embeddings."`, `max_tokens=200`, `temperature=0.0` →
deterministic, identical responses across runs). Two passes:

1. **5-run serial probe** on each image — captures cold-cache spike +
   warm steady-state latency.
2. **5-min sustained-load test on 26.05** — 4 concurrent workers issuing
   sequential `/v1/chat/completions` requests, stamping per-request
   wall-clock + completion-token count, sampling `nvidia-smi` every 30 s.
   25.11 was not load-tested; we were screening for *regressions* in
   26.05, not validating 25.11 (which has been running in production for
   weeks).

Test compose for 26.05 was a side-by-side
`docker-compose.yml.26.05` so the canonical compose stayed canonical
during evaluation. 26.05 image was already cached locally (pulled 5
weeks before the test — `c5c116c732f2`, 20 GB), so the test did not
incur a `docker pull` cost.

## Side effects surfaced during the bump

These are inline-documented in the promoted compose so future-me knows why:

1. **`VLLM_DISABLE_TORCH_COMPILE=1` env var no longer recognised on 26.05.**
   The GB10 Blackwell torch.compile hot-fix landed upstream; the env var
   emits `WARNING envs.py:1830 Unknown vLLM environment variable detected`.
   Dropped from this stack's compose. `coder-next/` keeps it because that
   stack remains on 25.11.

2. **26.05 picks up the model's `generation_config.json`** and applies its
   sampling defaults (`temp=0.7, top_k=20, top_p=0.8`):
   ```
   WARNING model.py:1437 Default vLLM sampling parameters have been
   overridden by the model's `generation_config.json`
   ```
   Our benchmarks override at request time so this doesn't affect
   measurements. Flag for any client that omits sampling params.

3. **Runtime MoE autotuner (the chipi/agentic-ai-homelab#1 obsoleter).**
   26.05 ships `flashinfer + trtllm` and auto-tunes the MoE kernel at
   startup instead of looking up a static JSON config:
   ```
   [AutoTuner]: Tuning trtllm::fused_moe::gemm1:   0%|  | 0/10 [00:00<?, ?profile/s]
   flashinfer.jit: [Autotuner]: Skipped 2 unsupported tactic(s) for trtllm::fused_moe::gemm1
   [AutoTuner]: Tuning trtllm::fused_moe::gemm1:  30%|███  | 3/10 [00:00<00:00, 18.57profile/s]
   ```
   The old `WARNING fused_moe.py:798 Using default MoE config` is gone.
   No `E=128,N=768,device_name=NVIDIA_GB10.json` to ship.

4. **NIXL warnings** (`NIXL is not available`) — multi-node RDMA library,
   irrelevant for single-GPU. Ignore.

5. **Cold-boot 7 % slower** (451 s → 481 s). Driven by per-shard load
   time going from ~23 s/it to ~26.5 s/it on 26.05. Absorbed by the
   `start_period: 30m` healthcheck buffer. Not concerning.

## What we deliberately did NOT do

Both saved by the runtime autotuner being shipped in 26.05:

- **No `benchmarks/kernels/benchmark_moe.py` sweep.** Would have produced a
  static JSON for the old `fused_moe.py` code path. 26.05 doesn't read
  the JSON; sweep wouldn't change runtime behaviour.
- **No upstream vLLM PR adding `configs/E=128,N=768,device_name=NVIDIA_GB10.json`.**
  Same reason: the mechanism changed; the file isn't loaded.
- **No Path-A before/after benchmark on podcast_scraper's prod corpus** for
  ticket #1. Skipped because there's no "before" / "after" pair to
  compare — the static-JSON code path doesn't exist in 26.05.

## Follow-up signal worth watching, not refiling today

The autotuner logs `Skipped 2 unsupported tactic(s)` during MoE-kernel
selection on GB10 — it isn't searching the entire kernel space and
could be leaving throughput on the table. If a future #996-style
measurement attributes throughput to autoresearch and surfaces an
obviously sub-optimal pattern, open a fresh scoped ticket for that
specific gap. The original tuning-via-static-JSON approach won't apply.

## Rollback procedure (if 26.05 surfaces a regression later)

The 25.11-py3 image is still cached on the DGX (`d33d4cadbe0f`, 14 GB).
Revert by editing `infra/vllm/autoresearch/docker-compose.yml`:

- `image: nvcr.io/nvidia/vllm:25.11-py3`
- Re-add `- VLLM_DISABLE_TORCH_COMPILE=1` under `environment:`

Then `gpu-mode-swap.sh idle && gpu-mode-swap.sh research`. ~7 min cold
boot. No other changes needed; weights, .env, prompts are all
image-independent.
